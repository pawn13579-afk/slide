여기는 발표자가 볼 수 있는 노트 영역입니다. 이 교육의 주요 목표는 참가자들이 마크다운을 사용해 효율적이고 재현 가능한 문서 및 슬라이드를 만드는 방법을 배우는 것입니다. 특히 Pandoc과 Quarto라는 강력한 오픈소스 도구를 중심으로 실무에 적용할 수 있는 지식을 전달하는 것을 목표로 합니다.

문서 작성 패러다임 변화

왜 코드 기반 문서 작성인가

  • 동적 콘텐츠 진화: 데이터와 코드 계산 결과를 포함
  • 과거 WYSIWYG: 수정이 어려웠던 정적 방식
  • 2000년대 웹의 등장: 콘텐츠와 디자인의 분리 시작
  • 2010년대 마크다운 + Pandoc: 진정한 싱글 소싱 시대 개척
  • AI 시대의 도래: 생성형 AI 결과의 체계적 문서화 필요성 증가
과거 문서 작성 방식은 수정이나 형식 변경에 많은 시간과 노력이 필요했지만, 이제 문서 작성은 단순히 텍스트를 입력하는 것을 넘어 동적인 데이터와 코드를 통합하는 방향으로 진화했습니다. 특히 AI가 생성한 인사이트를 검증 가능하고 재현 가능한 형태로 문서화하는 것이 중요해졌습니다.

마크다운과 싱글 소싱

콘텐츠, 디자인, 형식의 분리 및 통합

콘텐츠: 데이터 및 코드가 포함된 동적이고 재현 가능한 정보로 진화

디자인: 템플릿과 테마 기반의 체계적 디자인 시스템 구축이 주류

형식: 하나의 원본으로 다중 출력(HTML, PDF, Word 등) 생성

싱글 소싱: 하나의 원본 콘텐츠를 여러 형식에 적용하는 관리 방법

작업량 최대 80% 감소 및 콘텐츠 일관성 유지에 기여!

싱글 소싱은 콘텐츠를 한 번만 작성하고 다양한 출력 형식으로 자동 변환하여 중복 작업을 줄이고 효율성을 높입니다. 이는 현대 과학기술 문서나 비즈니스 글쓰기에서 콘텐츠 일관성을 유지하고 제작 효율성을 높이는 데 결정적인 역할을 합니다.

Pandoc: 만능 문서 변환 도구

다양한 마크업 포맷 간 상호 변환을 위한 Haskell 기반 명령줄 도구

마크다운을 AST(추상 구문 트리)로 변환 후, 원하는 형식으로 재변환

Pandoc은 마크다운 기반 문서 작성 워크플로우에서 가장 중요한 도구 중 하나입니다. 하나의 소스 파일에서 거의 모든 종류의 문서 형식으로 변환할 수 있는 범용성을 제공합니다.

Pandoc 주요 지원 출력 형식 (일부)

유형 출력 포맷 예시 명령어 옵션 (일부)
워드 프로세서 Word docx, ODT, RTF -t docx, -t odt
프레젠테이션 PowerPoint (pptx), reveal.js, beamer -t pptx, -t revealjs
웹/전자책 HTML, EPUB, Jupyter notebook -t html, -t epub
조판 시스템 LaTeX, ConTeXt, Typst -t latex, -t typst

Marp: 개발자 친화적 슬라이드 제작

마크다운 기반의 빠르고 가벼운 프레젠테이션 도구

  • 익숙한 마크다운 문법을 그대로 활용하여 작성
  • 세 개의 대시(---)로 슬라이드 구분
  • VSCode 확장을 통한 실시간 미리보기 및 내보내기 기능
  • PDF, PPTX, HTML 슬라이드로 변환 가능
Marp는 개발자에게 친숙한 Markdown 기반으로 빠르게 슬라이드를 작성할 수 있는 환경을 제공합니다. 복잡한 GUI 도구 대신 텍스트 기반으로 작업하여 버전 관리가 용이하다는 장점이 있습니다.

Marp 핵심 문법 및 예시

개념 문법/사용법 기능 설명
슬라이드 구분 --- 새로운 슬라이드 시작
전역 설정 --- marp: true theme: gaia --- 문서 제목, 테마, 페이지 번호 등 설정
이미지 크기 조절 ![w:500](URL) 이미지 너비를 500픽셀로 지정
배경 이미지 설정 ![bg left:40%](URL) 이미지를 배경으로, 왼쪽 40% 영역에 배치

Reveal.js: 인터랙티브 웹 슬라이드

HTML 및 JavaScript 기반의 강력한 프레젠테이션 프레임워크

  • Markdown 지원 및 자동 애니메이션(Auto-Animate)
  • PDF 내보내기 (`?print-pdf` 쿼리 스트링)
  • 발표자 모드 (s 키) 및 발표자 노트
  • 프래그먼트 (점진적 내용 표시)
  • 수식 렌더링 (MathJax/KaTeX)
Reveal.js는 웹 기술 기반으로 복잡한 디자인이나 사용자 정의 동작을 JavaScript API를 사용해 추가할 수 있습니다. Pandoc과 결합하면 마크다운 파일만으로도 전문적인 웹 기반 슬라이드를 생성하는 데 매우 유용합니다.

주요 슬라이드 제작 도구 비교

특징/도구 Pandoc Marp Quarto
주요 역할 범용 문서 변환 마크다운 슬라이드 저작 재현 가능한 출판 시스템

주요 사용자 일반 사용자, 개발자 개발자, 빠른 PT 제작자 데이터 과학자, 연구자
Pandoc은 변환 자체에 중점을 둔 엔진이라면, Marp는 간편한 슬라이드 작성 경험에, Quarto는 데이터 과학 및 학술 출판의 재현 가능성에 특화되어 있습니다. 사용 목적에 따라 적합한 도구를 선택하는 것이 효율적입니다.

Quarto: 차세대 R 마크다운

데이터 과학 출판을 위한 오픈소스 시스템

  • Pandoc을 근간으로 다중 언어(R, Python, Julia 등) 지원
  • R 마크다운의 성공 경험을 바탕으로 한 새로운 아키텍처
  • 싱글 소싱을 통해 다양한 출판 저작물(책, 웹사이트, 슬라이드) 생성
  • 코드와 데이터를 통합하여 오류 발생 위험 최소화
Quarto는 특히 데이터 과학 분야에서 여러 언어를 하나의 문서에서 통합하고 관리할 수 있도록 지원합니다. 이는 기존 R마크다운이 R 언어만 지원하던 한계를 넘어선 혁신적인 발전입니다.

싱글 소싱과 재현 가능성

하나의 소스 코드, 다양한 형식의 자동 변환


[ 단일 마크다운 소스 (.qmd) ]
         ↓
[ 코드 실행 및 통합 (Quarto Engine) ] (R, Python 등)
         ↓
[ 마크다운 변환 (Pandoc 입력값) ]
         ↓
[ 최종 출력 변환 (Pandoc) ]
         ↓
[ PDF, HTML, PPTX, Word 등 다중 출력 ]

데이터 변경 시 그래프나 분석 결과가 자동으로 갱신되는 '살아있는 문서'

싱글 소싱 방식은 콘텐츠를 한 번만 작성하여 수정이나 업데이트 시 일관성을 쉽게 유지할 수 있게 합니다. 이는 특히 데이터 기반의 문서에서 데이터가 변경되어도 문서가 자동으로 갱신되는 '살아있는 문서'의 시대를 열었습니다.

Quarto 문서의 세 가지 구성요소

메타데이터, 텍스트, 코드의 유기적 결합

  • 메타데이터 (YAML): 문서 제목, 작성자, 형식 등 전반적인 설정
  • 텍스트 (마크다운): 콘텐츠 구조와 의미에 집중한 글쓰기
  • 코드 (knitr/Jupyter): 데이터 분석, 그래프 등을 포함하는 실행 가능한 블록
Quarto 문서는 세 가지 주요 구성요소, 즉 YAML 메타데이터, Markdown 텍스트, 그리고 실행 가능한 코드 블록으로 이루어져 있습니다. YAML은 문서의 전반적인 설정을 명확하고 읽기 쉽게 관리할 수 있게 합니다.

format: pptx
code-fold: true
theme: [cosmo, brand.yml]
---

코드 통합: 동적 콘텐츠 생성

코드 덩어리(Code Chunk)를 이용한 데이터 통합

인라인 코드 예시

입력: `r pi`

결과: 파이 값은 3.1415927 이다.

코드 블록 예시 (R)


summary(mtcars[, 1:2])
코드 덩어리는 데이터 분석 문서의 핵심 요소입니다. 코드를 문서에 직접 포함하고 실행 결과를 반영함으로써 문서의 재현성과 정확성을 보장합니다.

수식 표현 및 렌더링

LaTeX 문법을 활용한 수학 공식 작성

  • 분수: \( \frac{1}{2} \) $\frac{1}{2}$
  • 극한: \( \lim_{x \to \infty} \exp(-x) = 0 \) $\lim_{x \to \infty} \exp(-x) = 0$
  • 적분 (블록):
    $$ \int_0^\infty \mathrm{e}^{-x}\,\mathrm{d}x $$
    $$ \int_0^\infty \mathrm{e}^{-x}\,\mathrm{d}x $$
데이터 과학이나 학술 문서에서 수학 수식은 필수적이며, LaTeX는 이를 표준적으로 표현하는 방법입니다. Pandoc과 Quarto는 이 LaTeX 문법을 활용하여 다양한 문서 형식에서 수식을 문제없이 표시할 수 있도록 지원합니다.

AI 시대의 콘텐츠 제작 파이프라인

AI와 쿼토의 결합을 통한 미래 문서 작성


[ 연구/분석 ] ---> [ 생성/처리 ] ---> [ 출력/검증 ]
  (데이터 연결      (코드 생성,        (시각화,
   버전 관리,        텍스트 생성)       표/수식,
   프롬프트 설계)                       품질 검증)
       ↑                ↑                ↑
       +----------------+----------------+
       |         AI 허브 (LLM)          |
       +--------------------------------+
AI는 문서 작성 과정을 완전히 자동화하는 것이 아니라, 창의적 통찰과 비판적 사고에 집중할 수 있도록 지능적인 어시스턴트 역할을 합니다. Quarto는 AI가 생성한 결과물을 체계적으로 관리하고 재현 가능한 문서로 통합하는 데 완벽하게 부합하는 도구입니다.

최종 요약: 마크다운 출판 시스템

  • Pandoc: 모든 변환의 핵심이 되는 범용 엔진
  • Quarto: 재현 가능한 동적 문서 출판을 위한 차세대 시스템
  • Marp/Reveal.js: 마크다운 기반의 효율적인 슬라이드 제작 도구
  • 싱글 소싱: 효율성과 일관성을 극대화하는 핵심 방법론
  • YAML 메타데이터: 문서의 모든 설정을 체계적으로 관리
이 교육을 통해 참가자들은 마크다운 기반의 현대적인 문서 작성 생태계가 왜 실무에서 필수적인지 이해했을 것입니다. Pandoc, Quarto, Marp를 활용하여 효율적이고 재현 가능한 문서를 제작하는 방법을 익히는 것이 중요합니다.
